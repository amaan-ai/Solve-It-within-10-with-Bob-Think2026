CitizenServicesApp — Java 8 legacy government portal (Java 21 upgrade candidate)

Upgrade CitizenServicesApp from Java 8 to Java 21 Agent Mode

Modernise this entire @CitizenServicesApp from Java 8 / Spring Boot 2.7 to Java 21 / Spring Boot 3.x. 1. Update pom.xml — bump Java to 21, Spring Boot to latest 3.x, migrate javax.* → jakarta.* dependencies. 2. Migrate source files using Java 21 features where they improve clarity: - Use records for simple data-carrier classes - Use pattern matching and sealed classes to simplify the status transition switch logic - Replace verbose anonymous classes and raw types with modern lambdas and generics 3. Replace any deprecated Spring Boot 2.x patterns with Spring Boot 3.x equivalents. 4. Show a before/after comparison for the 3 most impactful changes. 5. Confirm the upgraded application compiles — provide the run command.

Bonus: Run It & Add a Carbon Design Web UI Agent Mode

- Run this upgraded Java 21 CitizenServicesApp. - Add a web UI using Carbon Design System with: citizen lookup, service request submission form, and an officer dashboard showing pending requests with status update controls. - Open the web UI in a browser and test approving a service request end-to-end.
Take-Home Challenge: After Bob upgrades to Java 21, use this prompt to run the app and add a Carbon Design UI — then test the full citizen request lifecycle in the browser.
What This Demonstrates:
  • Java 8 → Java 21 in-place language and framework upgrade
  • Spring Boot 2.x → 3.x migration (javax → jakarta namespace, deprecated API removal)
  • Modern Java features: records, sealed classes, pattern matching
  • Compilation verification with a clear before/after diff
  • Bonus: Carbon Design UI and live end-to-end testing
